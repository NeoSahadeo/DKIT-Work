HTML5 Validation where Form Element Values are HARD-CODED (i.e. user not typing the value

When you click on the "submit" button, you shouldn't be able to get to the next page until:

  1. a radio button has been clicked (or checked);
  2. a selection list option, that has a non-empty value, has been chosen.

HTML5 Validation of RADIO BUTTONS:
(1) set each of them as "required"

HTML5 Validation of SELECTION LIST:
(1) set <select> as "required"
(2) first <option> should be informational text and value="" (i.e. blank value)
